# Plan-driven sessions

`PLAN.md` is the session's source of truth for what to work on.

## When reading PLAN.md

- Treat it as your task list — identify the next uncompleted item and work on it
- If the plan has a `## Next` section, start there

## When updating PLAN.md

- Preserve the original ticket context at the top (title, description, URL)
- Mark completed items with `[x]` or strikethrough
- Add session-by-session updates under `## Progress`
- Keep the next task clearly identified in a `## Next` section
- Note any blockers, decisions made, or context that future sessions need
