# Database Architect Agent Memory

## Import Patterns

- PR state queries (`getPRState`, `listPRStates`, `upsertPRState`) live in `@tribunal/github/pull-requests/state/state` and require `GithubServiceContext`.
- Web app service files use `import { db } from '$lib/server/database'` and import operators from `drizzle-orm` directly.

## Schema Notes

- The schema is intentionally minimal: auth (`user`, `session`, `auth-account`, `oauth-connection`), GitHub (`github-installation`, `github-installation-repository`, `github-webhook-delivery`, `webhook-event`), and repository/PR (`repository`, `pull-request-state`, `pull-request-trigger`). There are no sandbox, goal, project, or workspace tables.
- `pullRequestState.repositoryId` is `bigint` with `mode: 'number'` (matches GitHub ID pattern).
- `pullRequestState` uses timestamps without timezone.
- Unique index on `(repositoryId, prNumber)` for `pullRequestState`.

## SvelteKit Route Patterns

- API routes use `+server.ts` (NOT `index.ts`).
- Import `RequestHandler` from `./$types` (generated by SvelteKit).
- Run `bunx svelte-kit sync` after creating new routes to generate `$types`.
- Import `json` from `@sveltejs/kit`.
